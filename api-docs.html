API Documentation - Train Booking System

API Endpoints

1. Get Seat Availability

Endpoint: /api/v1/getSeatAvailability

Method: GET

Description: Fetches available trains between a source and destination.

Request Body

{
  "source_station": "Station A",
  "destination_station": "Station B"
}

Response (on success):

{
  "success": true,
  "message": "Successfully fetched train data between given source and destination",
  "data": [
    {
      "train_id": 1,
      "train_number": 12345,
      "source_station": "Station A",
      "destination_station": "Station B",
      "total_seats": 100,
      "available_seats": 50
    }
  ]
}

2. Book a Seat

Endpoint: /api/v1/bookSeat

Method: POST

Description: Books a seat for a user on a specific train.

Request Body

{
  "train_number": 12345,
  "source_station": "Station A",
  "destination_station": "Station B"
}

Response (on success):

{
  "success": true,
  "message": "Successfully booked the ticket",
  "booking_status": "Confirmed"
}

3. Add Train

Endpoint: /api/v1/admin/add-train

Method: POST

Description: Adds a new train to the system (Admin Only).

Request Body

{
  "train_number": 54321,
  "source_station": "Station A",
  "destination_station": "Station C",
  "total_seats": 150,
  "available_seats": 150
}

Response (on success):

{
  "success": true,
  "message": "Train added successfully"
}

4. User Registration

Endpoint: /api/v1/register

Method: POST

Description: Registers a new user.

Request Body

{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "password123"
}

Response (on success):

{
  "success": true,
  "message": "User registered successfully"
}

5. User Login

Endpoint: /api/v1/login

Method: POST

Description: Logs in an existing user.

Request Body

{
  "email": "john@example.com",
  "password": "password123"
}

Response (on success):

{
  "success": true,
  "message": "User logged in successfully",
  "token": "your-access-token-here"
}
